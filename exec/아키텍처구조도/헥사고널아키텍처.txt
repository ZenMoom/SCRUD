헥사고날 아키텍처

├── src
│   ├── main
│   │   ├── java/com/example/app
│   │   │   ├── domain              ← 비즈니스 규칙 (엔티티, 인터페이스)
│   │   │   ├── application         ← 유스케이스 (비즈니스 흐름)
│   │   │   ├── adapter
│   │   │   │   ├── in              ← 외부 입력 (Controller, Consumer)
│   │   │   │   └── out             ← 외부 출력 (Repository, API Client 등)
│   │   │   └── config              ← 설정 클래스
│   └── test

의존성 방향이 내부(domain)로만 향함
외부 시스템(I/O)을 adapter로 분리해 test 용이성 증가
복잡한 도메인 비즈니스에 적합
